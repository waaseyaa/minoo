# WP06 — Bimaaji MCP verification

## Objective

Confirm the `bimaaji` MCP server still ships and registers in Claude Code after upstream alpha.164 removed the *spec* MCP servers. `composer bimaaji-mcp-install` must exit 0 and `mcp__bimaaji__*` tools must resolve.

## Context

- alpha.164 release notes: "feat(#1347): adopt Spec Kitty and remove spec MCP servers". The "spec" MCPs were the `waaseyaa_*` and `minoo_*` servers used for retrieving specs — those are now spec-kitty-shaped.
- `waaseyaa/bimaaji` is a *different* MCP — Minoo's bridge to Anishinaabemowin tooling. It's a separate package and should be unaffected.
- `.claude/settings.json` registers both `minoo` (`mcp/server.js`) and `bimaaji` (`vendor/waaseyaa/bimaaji/mcp/server.js`).
- CLAUDE.md: "After `composer install`, run `composer bimaaji-mcp-install` (or rely on `post-create-project-cmd`) so both MCP servers have Node deps."

## Branch Strategy

Direct work on `main` (after WP01). No worktree. This WP produces a research log.

## Subtasks

### T025 [P] — Run `composer bimaaji-mcp-install`; verify exit 0

**Purpose**: Confirm the MCP install script still works against alpha.171.

**Steps**:
1. Run: `composer bimaaji-mcp-install 2>&1 | tee /tmp/bimaaji-install.log`.
2. If exit 0 and Node deps are present in `vendor/waaseyaa/bimaaji/mcp/node_modules/`, success.
3. If failure, read the error:
   - "command not found" → script may have been renamed in alpha.164–171 composer-script changes; check `composer.json` `scripts` section and `vendor/waaseyaa/bimaaji/composer.json`.
   - "package.json missing" → vendor path may have moved; locate via `find vendor/waaseyaa/bimaaji -name package.json`.
   - "npm error" → Node version compatibility.

**Validation**:
- `composer bimaaji-mcp-install` exits 0.
- `ls vendor/waaseyaa/bimaaji/mcp/node_modules/` shows installed packages.

### T026 [P] — Verify `.claude/settings.json` still registers minoo + bimaaji

**Purpose**: Confirm the MCP config wasn't accidentally clobbered by `spec-kitty init` (the init step we ran earlier added `.claude/` to .gitignore).

**Steps**:
1. Read `.claude/settings.json`.
2. Confirm `mcpServers.minoo` and `mcpServers.bimaaji` entries exist.
3. Confirm the `command`/`args` for each points to a file that exists on disk.
4. If either is missing, restore from `git show HEAD:.claude/settings.json` and re-add the missing entry.

**Validation**:
- Both MCP server entries present and pointing to existing files.
- `git status .claude/settings.json` is clean (or contains only an intentional restore).
