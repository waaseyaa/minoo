# WP02 — Integration tests: alpha.158 kernel-subclass ban

## Objective

Reconcile Minoo's integration tests with the alpha.158 framework change that drained `MinimalTestKernel` and forbids named kernel subclasses anywhere under `tests/**`. Integration tests must use whatever sanctioned boot path the framework now exposes, without losing the in-memory SQLite kernel-boot pattern.

## Context

- Per CLAUDE.md gotchas: "Integration tests boot HttpKernel with reflection (`boot()` is protected), use `putenv('WAASEYAA_DB=:memory:')` for in-memory SQLite."
- alpha.158 release notes mention: "drain MinimalTestKernel" and "forbid named kernel subclasses in tests/**".
- The reflection-based boot pattern may itself be the offender, OR the offender may be a named subclass somewhere in `tests/App/Integration/` that we've been carrying forward.
- Framework MCP is available (`waaseyaa_get_spec testing` should describe the new sanctioned pattern).

## Branch Strategy

Direct work on `main`. Depends on WP01 (composer bump must be in place to test against alpha.171). No worktree.

## Subtasks

### T007 — Inventory named kernel subclasses in tests/**

**Purpose**: Find every place in `tests/**` that subclasses or names a Kernel-derived class, so we know the exact change surface.

**Steps**:
1. Run: `grep -RIn -E "extends (HttpKernel|ConsoleKernel|MinimalTestKernel|TestKernel)" tests/`.
2. Run: `grep -RIn -E "class \w+Kernel" tests/` to catch any other Kernel-named class definitions.
3. Run: `grep -RIn "MinimalTestKernel" tests/ vendor/waaseyaa/` to confirm what the framework now offers (or doesn't).
4. Capture the inventory in a brief note (inline comment in the WP commit, or `kitty-specs/upgrade-waaseyaa-alpha-171-01KQTDC2/research/kernel-subclass-inventory.md`).

**Validation**:
- The inventory enumerates every offending file with line numbers.

### T008 — Refactor integration tests to sanctioned boot path

**Purpose**: Replace the banned pattern with whatever the framework now blesses.

**Steps**:
1. Read `vendor/waaseyaa/testing/` (if exists) and `vendor/waaseyaa/foundation/` to find the new sanctioned boot helper. Likely candidates:
   - A `bootForTesting()` static factory on `HttpKernel`.
   - A trait like `BootsHttpKernel` or `IntegrationTestCase` base class.
   - A test container builder.
2. Use `waaseyaa_get_spec testing` MCP if available for the canonical guidance.
3. For each file in the T007 inventory, refactor to the new pattern. Preserve:
   - The in-memory SQLite mode (`putenv('WAASEYAA_DB=:memory:')`).
   - The 3-levels-up `dirname(__DIR__, 3)` to project root.
   - Any `#[CoversNothing]` or `#[CoversClass]` attributes.
4. If the old pattern is just *fine* and the ban only affects subclass-by-name (not reflection), the simplest fix is to verify there's no offending subclass and call it done.

**Files**: every file in T007 inventory (most likely `tests/App/Integration/*Test.php`).

**Validation**:
- `grep -RIn -E "extends (HttpKernel|ConsoleKernel|MinimalTestKernel|TestKernel)" tests/` returns no results (or only sanctioned base classes from the framework).
- `php -l` on each refactored file returns "No syntax errors".

### T009 — Run `--testsuite MinooIntegration`; capture failures, fix, repeat

**Purpose**: Achieve a green integration suite.

**Steps**:
1. Run: `./vendor/bin/phpunit --testsuite MinooIntegration 2>&1 | tee /tmp/integration.log`.
2. If failures, read each one and decide:
   - Genuine framework breakage: report upstream, file a follow-up issue, but do NOT block this WP unless the breakage is widespread.
   - Minoo-side test bug: fix in this WP.
   - Refactor regression from T008: fix in this WP.
3. Iterate until exit 0.

**Validation**:
- `./vendor/bin/phpunit --testsuite MinooIntegration` exits 0.
- Test count and assertion count are at parity with pre-upgrade baseline (or higher).

## Risks

- **Reflection ban**: If alpha.158 also bans the `(new ReflectionMethod(HttpKernel::class, 'boot'))->setAccessible(true)` pattern, the refactor surface widens. Mitigation: read the framework testing spec via MCP before assuming.
- **Sanctioned helper gap**: The framework may not yet offer a clean replacement, in which case file an upstream issue and pick the least-bad workaround for this mission (e.g., a private trait local to Minoo's tests).
