# WP07 — Local smoke tests

## Objective

Verify the public surface renders end-to-end on the upgraded framework before merging to `main`. Five canonical routes must return HTTP 200 with body > 1KB and `<title>` present. Past framework jumps (alpha.75 → alpha.107) silently broke `Response::send()` and produced zero-byte 200s in production — this WP exists specifically to catch that class of failure before deploy.

## Context

- CLAUDE.md gotcha: "Verify production with body size, not HTTP status. A `curl -I` returning 200 does NOT prove the app is alive. Always `curl -sS -o file -w '%{http_code}/%{size_download}'` and spot-check a `<title>` tag."
- `public/index.php` MUST always call `$response->send()` regardless of SAPI — verify this didn't regress.
- Use `0.0.0.0` not `localhost` on WSL2 so Windows browsers can reach the server.

## Branch Strategy

Direct work on `main` (after WP02–WP04). No worktree.

## Subtasks

### T028 — Boot local server

**Purpose**: Get a local PHP server running against the upgraded framework.

**Steps**:
1. From repo root: `php -S 0.0.0.0:8080 -t public public/index.php > /tmp/php-server.log 2>&1 &`.
2. Capture the PID for later cleanup.
3. Wait 2 seconds, then verify: `curl -sS -o /dev/null -w '%{http_code}\n' http://localhost:8080/`. Should be 200 (or a redirect for anonymous → public).

**Validation**:
- Server boots without fatal errors in `/tmp/php-server.log`.
- A baseline `curl /` returns a non-error status.

### T029 — Smoke `/` (anonymous public homepage)

**Purpose**: Verify the public homepage renders.

**Steps**:
1. `curl -sS -o /tmp/homepage.html -w "code=%{http_code} size=%{size_download}\n" http://localhost:8080/`.
2. Assert: `code=200`, `size > 1024`.
3. Assert: `grep -c '<title>' /tmp/homepage.html` is 1.
4. Spot-check: `grep -c 'Minoo\|Anishinaabe\|Sagamok' /tmp/homepage.html` is >= 1 (any expected content marker).

**Validation**:
- 200 response, body > 1KB, `<title>` present, expected content marker found.

### T030 — Smoke `/feed` (authenticated)

**Purpose**: Verify the authenticated social feed renders.

**Steps**:
1. Anonymous `/feed` should redirect to `/` (302). Verify: `curl -sS -o /tmp/feed-anon.html -w 'code=%{http_code}\n' http://localhost:8080/feed`. Expect 302 to `/`.
2. To test authenticated view, either:
   - (a) Skip if no test user is seeded — note in T034 as "manual verification needed".
   - (b) Use `php scripts/populate_engagement.php` to seed users, then construct a session cookie via the auth flow.
3. If (b), `curl -sS -b cookies.txt -o /tmp/feed.html -w "code=%{http_code} size=%{size_download}\n" http://localhost:8080/feed` after login.

**Validation**:
- Anonymous /feed returns 302.
- Authenticated /feed (if testable) returns 200 with body > 1KB.

### T031 — Smoke `/games`, `/games/shkoda`, `/games/crossword`, `/games/agim`

**Purpose**: Verify all 3 games + the hub render.

**Steps**:
1. For each route, run: `curl -sS -o /tmp/<slug>.html -w "code=%{http_code} size=%{size_download} url=%{url_effective}\n" http://localhost:8080/<route>`.
2. Each must be 200 with body > 1KB and `<title>` present.
3. Spot-check `/games/shkoda` for "fire" or "shkoda" content; `/games/crossword` for puzzle grid markup; `/games/agim` for number-counting UI.

**Validation**:
- All 4 routes return 200, body > 1KB, expected content markers found.

### T032 — Smoke a community detail page

**Purpose**: Exercise the NorthCloud client adapter (which alpha.171 may have touched indirectly).

**Steps**:
1. Pick a known-seeded community: `bin/waaseyaa db:query "SELECT slug FROM community LIMIT 1"` or use `sagamok` as the canonical Sagamok Anishnawbek slug.
2. `curl -sS -o /tmp/community.html -w "code=%{http_code} size=%{size_download}\n" http://localhost:8080/communities/sagamok`.
3. Expected: 200 with body > 1KB.
4. The NC client may try to reach `localhost:8050` (source-manager) — that may not be running locally. Acceptable degraded behavior: page renders without leadership/band-office data, with a graceful empty state. Unacceptable: 500 or fatal exception.

**Validation**:
- 200 response, body > 1KB.
- Page renders; absence of NC data is acceptable degradation.

### T033 — Smoke `/elder-support`

**Purpose**: Verify Elder Support request form renders.

**Steps**:
1. `curl -sS -o /tmp/elder.html -w "code=%{http_code} size=%{size_download}\n" http://localhost:8080/elder-support`.
2. Expected: 200 with body > 1KB.
3. Spot-check: form action attribute present, CSRF token (if applicable) rendered.

**Validation**:
- 200 response, body > 1KB, form markup present.

## Definition of Done

- [ ] All 5 smoke routes return 200, body > 1KB, `<title>` present.
- [ ] No fatal exceptions in `/tmp/php-server.log`.
- [ ] Smoke results documented and committed.
- [ ] Local server terminated.

## Risks

- **Zero-byte 200s**: The exact failure mode this WP exists to catch. If any route returns 200 with body < 100 bytes, STOP and investigate — likely a `Response::send()` regression.
- **NC client unreachable**: Local dev typically doesn't run source-manager on `localhost:8050`. Don't fail the WP for this; document and move on.
- **Authenticated routes**: `/feed` requires a session. If no test user is seeded, mark as "manual verification needed" rather than blocking the WP.

## Reviewer Guidance

- Confirm body sizes are believable (a real homepage is several KB, not exactly 1024 bytes).
- Confirm any "manual verification needed" notes have follow-up plans.
