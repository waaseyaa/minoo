# WP01 — EntityCommunityProvider cluster (Group, Leader, Contributor)

## Objective

Migrate the 3 community-bundle entities owned by `EntityCommunityProvider`
from the deprecated `Waaseyaa\Entity\Community\HasCommunityInterface`
marker to the framework's explicit `tenancy: ['scope' => 'community']`
declaration on the `EntityType` constructor.

## Context

This is the first of 3 WPs in the mission. WP01 establishes the pattern
the other 2 WPs follow. Read [spec.md](../spec.md), [plan.md](../plan.md),
and [data-model.md](../data-model.md) before starting; especially:

- `research.md` Decision 1 — the framework's tenancy contract is
  `array{scope: string}|null` on `EntityType`'s constructor, validated
  strictly. See `vendor/waaseyaa/entity/src/EntityType.php` lines 52–113.
- `research.md` Decision 2 — single-pass: each entity's marker removal
  AND `tenancy:` addition land in the same commit (constraint C-002).

### T002 — Add `tenancy:` to provider registrations

**Purpose**: Add the explicit tenancy declaration to each of the 3
`new EntityType(...)` calls.

**Steps**:

1. Open `src/Provider/Entity/EntityCommunityProvider.php`.
2. For each of the 3 `EntityType` constructions identified in T001,
   add `tenancy: ['scope' => 'community']` as a named argument. Place
   it after `entityKeys:` for readability:

   ```php
   $this->entityType(new EntityType(
       id: 'group',
       label: 'Group',
       entityKeys: ['id' => 'gid', 'uuid' => 'uuid', 'label' => 'name'],
       tenancy: ['scope' => 'community'],
       // ...remaining args...
   ));
   ```

3. **Do not** modify any other named argument. Do not reformat
   surrounding code. The diff for this subtask should be exactly 3
   one-line additions inside 3 `new EntityType(` calls.

**Validation**:

- [ ] PHPUnit will fail later if the array shape is wrong; the framework throws `InvalidArgumentException` for any non-`['scope' => 'community']` tenancy.
