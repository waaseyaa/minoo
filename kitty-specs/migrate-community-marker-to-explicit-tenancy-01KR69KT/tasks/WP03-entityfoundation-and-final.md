# WP03 — EntityFoundationProvider (Post) + final reconciliation

## Objective

Migrate the last marker-tagged entity (`Post`) and run mission-level
reconciliation to confirm the bulk-edit promise: zero remaining
`HasCommunityInterface` references in `src/`. This WP closes the
umbrella issue.

## Context

WP01 and WP02 have already merged. Six of the 7 entities are migrated.
WP03 finishes the seventh and validates the mission's success criteria
(SC-001 through SC-004). Read [spec.md](../spec.md), [plan.md](../plan.md),
[data-model.md](../data-model.md), and the merged WP01/WP02 PRs before
starting.

Note that Post is documented in `data-model.md` as owned by
`EntityFoundationProvider`, but this is provisional — Post could equally
plausibly live in `EntityFeedProvider`. T017 verifies this before any
edits.

### T017 — Verify Post's owning provider

**Purpose**: Confirm Post is registered in `EntityFoundationProvider`
(and not e.g. `EntityFeedProvider`). This is the highest-risk
verification step in the mission.

**Steps**:

1. Grep for Post's bundle key across all entity providers:
   ```bash
   grep -rn "new EntityType\(" src/Provider/ | grep -E "'post'"
   ```
2. If the hit is in `EntityFoundationProvider.php`, proceed to T018
   with no changes.
3. If the hit is in a different provider:
   - **Stop.**

     point at the actual provider file.
   - Update `data-model.md` row 7 to reflect the actual provider.
   - Run `spec-kitty agent mission finalize-tasks --validate-only`
     to confirm no overlap.
